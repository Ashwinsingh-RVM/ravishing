How Does This Model Work?

A step-by-step guide to how we estimate the fleet needed to collect DRS containers from HoReCa establishments across Goa. Scoped to FY27 (Apr-26 to Mar-27) — the first year of DRS, covering alcohol containers only.

1

Identifying HoReCa Establishments

▸

We start with 15,675 Google Places records across Goa — every restaurant, bar, hotel, cafe, guesthouse, beach shack, and resort.

Since FY27 is alcohol-only under DRS, we filter for establishments that serve or are likely to serve alcohol:

Final set: 8,313 HoReCas — Restaurants (5,603), Hotels (1,816), Bars (634), Beach Shacks (159), Cafes (87), Resorts (12), Others (2).

2

Bottom-Up Waste Generation Estimates

▸

Each HoReCa type has a baseline daily generation estimate (bottles/day). A size multiplier adjusts for establishment size: Large (2x), Medium (1x), Small (0.5x), Micro (0.25x).

A Medium Bar: 100 btl/day x 1.0 (medium) = 100 btl/day A Large Restaurant: 70 btl/day x 2.0 (large) = 140 btl/day A Small Cafe: 10 btl/day x 0.5 (small) = 5 btl/day

These are relative weights — the absolute numbers are calibrated by PnL data (Step 3).

3

DRS Return Units — Anchoring to Reality

▸

The PnL model provides exact DRS return units per month per material. We apply HoReCa share (currently 50%) to get the daily units HoReCa handles.

PnL says: 1.76 Cr DRS return units in December 2026 Daily return units: 1.76 x 1,00,00,000 / 30 = 5,86,667 units/day HoReCa share (50%): 5,86,667 x 0.50 = 2,93,333 units/day Calibration factor = PnL total / bottom-up total Scales all estimates to match PnL — bottom-up provides relative distribution, PnL provides absolute scale.
4

Assign HoReCas to Warehouses

▸

Goa has 5 warehouses strategically placed from South to North:

Margao South (South Goa) | Cortalim (South-Central) | Calangute North (Tourist Belt) | Mapusa North (Interior) | Arambol (Far North)

Each HoReCa is assigned to its nearest warehouse using haversine distance x road factor (1.4x). Calangute North gets the largest cluster at ~40%.

5

Pickup Scheduling

▸

HoReCas are sorted by daily volume and assigned pickup frequency buckets:

Daily (20%) — high-volume bars, large restaurants
Alternate days (30%) — medium restaurants, hotels
Twice weekly (30%) — smaller restaurants, cafes
Weekly (20%) — guesthouses, homestays, low-volume places

Expected daily pickups = (Daily x 1.0) + (Alternate x 0.5) + (Twice weekly x 2/7) + (Weekly x 1/7)
6

Fleet Sizing (with Volumetric Weight)

▸

Daily pickups are split by road type, then each road type uses appropriate trucks:

30% narrow streets -> Tata Ace (1T) + Bolero (1.5T)
45% medium roads -> Intra V50 (2T)
25% main roads -> Intra V70 (2.5T)

Stops per trip is capped by the 5-hour pickup window (~18-20 stops per trip). 1 trip per vehicle per day — the window is fully consumed by collection. Trucks fill by volume, not just weight — a glass-heavy mix has a volumetric factor of ~1.6x.

Trips needed = ceil(pickups / stops_per_trip) Vehicles needed = trips (1 trip per vehicle) Daily cost = vehicles x daily_cost_inr per fleet type
7

The Result

▸

For each warehouse: vehicle count by type, daily + monthly cost from actual vendor quotations, fleet mix optimized by road type distribution.

The Simulation tab lets you tweak any parameter and re-run instantly.

8

Sensitivity Analysis — What Matters Most?

▸

Key drivers ranked by median impact across 12 months:

1. Pickup Frequency (~18.8%) — Directly changes how many daily stops trucks make
2. Material Mix (~18.2%) — Heavier mix (more glass) fills trucks faster
3. HoReCa Share (~13.9%) — What fraction of total DRS returns flow through HoReCa
4. Total Volume (~9.8%) — Overall return unit projection accuracy
5. Road Distribution (~3.0%) — Mix of narrow vs main road routes
6. Operational Efficiency (~0%) — Scan rate and setup time have minimal impact
